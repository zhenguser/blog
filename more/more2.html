# 微服务电商项目

# 一.微信服务

# 二维码生生接口(WeiXinQrCodeServiceImpl)

# getQrUrl

根据userId生成待参数连接的二维码,提供给会员服务登入接口使用。

  1. 根据appid 获取对应的 WxMpQrcodeService
  2. 利用WxMpQrcodeService,根据userId生成二维码的ticket
  3. 前端vue根据ticket拼接链接地址，生成二维码，用户扫描二维码触发关注事件（SubscribeHandler）或者扫描事件（ScanHandler）

# 微信关注事件（SubscribeHandler）

  1. 获取用户基本信息：userId、openId
  2. 通过WxMpServiceManage调用会员服务的MemberInfoService服务，查询该openId是否已经关联了账号
  3. 若没关联，则调用会员服务的MemberInfoService服务，关联账号，否则不做关联操作

# 微信扫码事件（ScanHandler）

  1. 通过WxMpServiceManage调用会员服务的MemberInfoService服务，查询该openId是否已经关联了账号
  2. 若没关联，则调用会员服务的MemberInfoService服务，关联账号，否则不做关联操作

# 登入消息模板(WeChatLoginTemplateImpl)

发送登入提醒

# 二.会员服务

# 注册接口（MemberRegisterServiceImpl）

# register

  1. 参数验证
  2. 查询数据库检查该手机号码是否存在
  3. 密码加密 MD5 +加盐值，插入数据库中

TIP

盐值生成： 1.盐值采用时间挫或其他，存入数据库中，每个用户的盐值不一样，安全性高 2.在配置中心定义一个常量用于盐值，但是每个用户的盐值都是一样的，安全系数低 3.使用手机号作为盐值，会导致用户更改手机号后，登入失败

# 登入接口（MemberLoginServiceImpl）

前端vue如何绑定会话信息 前端vue使用ajax调用后端登陆接口，获取令牌存放到缓存中，每次在调用其他的接口都会传递该接口。
登陆成功之后返回令牌给客户端保存，客户端后期使用该令牌绑定会话信息

# login

  1. 参数验证 渠道参数可放在网关中验证
  2. 密码采用MD5加密，根据密码查询我们的数据库，获取用户信息
  3. 生成token （key:采用项目前缀+uuid value: userId） key:
    有效期 生成临时且唯一,采用项目前缀+uuid。 value:
    不要放入用户信息、手机号，这些信息都是可变的。可以放userId,拿到token可以根据value中的userId查询数据库，获取用户信息。
  4. 通过AsyncLoginLogManage异步日志记录 (1)根据当前的渠道+userid+可用 查询 该用户是否已经登陆过 (2)如果没有查询到记录,创建token存入redis中，并插入最新的token到数据库中 (3)如果有记录，将数据库记录改为不可用，，并删除redis中的token，最后创建新的token存入redis中，并插入最新的token到数据库中 (4)调用微信接口发送消息模版 通过weChatLoginTemplateFeign调用微信服务WeChatLoginTemplate

如何获取客户端真实ip地址 因为从request中获取的IP地址是gateway的，gateway中的IP地址是nginx中转发的,nginx做了代理，所以不是真实IP地址 解决办法： Nginx nginx可以获取真实的客户端ip地址，存放到请求中 请求头中设置该参数 X-Real-IP value vue为客户端真实的ip地址

如何实现唯一登入 第一次登陆生成令牌 token
第二次登陆之后 之前的token从Redis删除掉。
在nacos中配置渠道列表
一个渠道对应只能生成一个token
流程： 1.查询我们的登陆记录表 条件有 渠道+userid+当前token是否可用 2.没有查询登陆记录情况下说明 该渠道对应的该用户没有登陆过直接向该表插入一条日志记录 3.如果查询到有登陆记录的情况下，说明该账户在登陆状态，将数据库中的状态变为不可用。从redis中移除原来令牌，插入最新的登陆日志信息 客户端实现: Ajax定时开启 每2s时间检查当前自己的token是否有失效

扫码关注 原理： 1.判断该用户是否关注公众号 2.如果没有关注公众号，调用微信接口生成二维码连接地址（二维码中含有参数userId，和微信用户的openId） 如何关联： 当用户扫描二维码时，会将二维码中的userId传递到微信回调接口，此时可以获取到关联用户的userId 1.先根据该openId查询是否关注公众号 2.获取userId，查询用户信息，修改openId

实现： 1.生成带参数的链接,传递用户的userid，获取微信展示的二维码链接 微信服务提供接口(WeiXinQrCodeServiceImpl getQrUrl)： 根据userId生成待参数连接的二维码（vue直接传递userId不安全） 解决方案： vue调用会员服务接口传递token获取带参数链接的二维码 (1)根据token获取userId (2)调用微信服务接口,根据userId生成待参数连接的二维码 2.用户打开手机二维码识别该二维码时，则会将该参数(userId,openId)通知给微信的回调接口 3.回调接口获取该userid，判断该用户是否有关联过openid，没有关联则关联userid

微信端: 如果用户没有关注微信公众号，不会推送扫码事件，改为推送关注事件(SubscribeHandler), 如果该用户已关注微信公众号，推送扫码事件(ScanHandler)

用户授权成功后，将对应的openId与数据库中的账户关联 原理：

  1. 根据用户授权的的openId查询数据库是否关联账户，没有关联账户则跳转到关联页面

oauth2.0流程：

  1. 用户选择账户授权
  2. 获取到授权码code
  3. 根据授权码获取accessToken
  4. 根据accessToken获取用户的openid

# 三.会员定时任务服务

定时任务数量大的情况下如何处理 1.采用多线程，但是非常消耗单台哦服务器的内存 在集群中，如何保证定时任务只会触发一次

  1. 将业务逻辑和定时任务逻辑完全分开部署，只对业务逻辑实现集群，不对我们的定时任务逻辑集群；
  2. 对我们Jar包加上一个开关，项目启动的时候读取该开关 如果为true的情况下则加载定时任务类，否则情况下就不加载该定时任务类；
  3. 在数据库加上一个主键能够创建成功，则触发定时任务，否则就不触发定时任务；
  4. 分布式锁实，只要jar能够拿到分布式锁就能够执行定时任务，否则情况下不执行； 以上的方案都是属于规模比较小的项目，在微服务架构中应该采用分布式任务调度平台。 分布式任务调度平台
  5. 定时任务的项目将自己服务器地址注册到分布式任务调度平台中；
  6. 我们的定时任务的规则都会在“分布式任务调度中心”触发；
  7. 当我们的任务调度中心触发定时任务规则时，读取“定时任务项目服务器集群地址”，取一个定时任务地址，调度中心向定时任务发一个通知根据java反射机制执行定时任务项目。

订单调用支付服务预插入支付信息

  1. 验证参数

# 支付接口

  1. 验证参数
  2. 根据渠道id查询到详细渠道的信息
  3. 根据beanid 从Spring容器中找到策略类执行
  4. 根据payToken查询到支付ID
  5. 根据支付ID获取我们的预支付的参数的信息
  6. 统一返回html表单给客户端

# 异步回调接口

  1. 根据渠道id查询渠道信息
  2. 获取模板的baenid
  3. 直接从Spring容器上下文获取模板类 3.1 验证签名 3.2 记录日志 注意问题：一定要是异步，提高快速响应给支付宝，避免重试造成的幂等性问题 3.3 调用子模板类的子类 3.3.1 根据支付订单号码查询数据库 ，是否已经支付过 3.3.2 判断支付的金额 是否一致 3.3.3 修改为已经支付状态 3.3.4 调用积分服务接口增加积分 分布式失误的问题
